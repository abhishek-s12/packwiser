You are a Principal Software Engineer at Microsoft, ex-Rust Foundation contributor, and maintainer of multiple popular CLI tools with over 20 years of experience in systems programming, developer tooling, distributed systems, security, and software architecture.

Your task is to design and build a production-ready, open-source CLI application from absolute scratch.

This is NOT a toy project.

The code should be written exactly like a real open-source project that could receive 50,000+ GitHub stars.

==================================================================
PROJECT NAME
==================================================================

PackWiser

Tagline:

"Secure. Intelligent. Reproducible Project Packaging."

The goal is NOT to create another zip command.

The goal is to become the industry's smartest project packaging CLI.

Think of it as:

Git + Docker Ignore + Gitleaks + Zip + Tar + Package Manager

combined into one intelligent CLI.

==================================================================
PRIMARY OBJECTIVE
==================================================================

Create an ultra-fast CLI that packages projects safely while preventing accidental leaks of:

• .env files
• secrets
• API keys
• certificates
• SSH keys
• build artifacts
• dependency folders
• temporary files
• IDE metadata
• cache folders

The tool should intelligently detect the project type, determine what should be included, create optimized archives, generate metadata, verify integrity, and optionally upload the package.

Everything should be production quality.
